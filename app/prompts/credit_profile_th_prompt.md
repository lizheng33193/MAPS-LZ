## 输入字段语义（risk_feature_labels 11 项）

| 字段 | 中文释义 |
|---|---|

## 哨兵值语义（sentinel_values）

下列值不是数值缺失，请按以下语义解读，不要当成 0 或低分参与计算：

- `liveness_score = "无活体分"` → 用户尚未进入活体检测环节，**不要**判定为低活体可信度，应明示「未进入活体环节」
- `max_yuqi_days = "无逾期"` → 历史无任何逾期记录（**积极**履约信号），必须在解读中明示
- `rule_hit_多头规则拦截 = "无记录"` → 多头借贷规则**未触发**判定，**不要**默认按命中处理
- `rule_hit_逾期未结清拦截 = "无记录"` → 逾期未结清规则**未触发**判定，**不要**默认按命中处理

## 输出要求（5 段 markdown，二级标题严格如下）

### 1. 身份核验

- 解读 `liveness_score`
- 哨兵值「无活体分」需明示「用户未进入活体环节」，不要打低分
- 数值型分数请给出对应风险层级与简短建议

### 2. 申请行为

- 综合解读 `apply_7d_num` / `apply_refuse_num` / `cashloan_app_num`
- 重点判断**多头借贷**倾向，给出强/中/弱判断与依据
- 当近期申请被拒次数偏高时，结合多头信号联动解释

### 3. 还款履约

- 解读 `finished_assets_num` / `max_yuqi_days`
- 哨兵值「无逾期」必须在结论中明示为**积极信号**
- 当存在逾期天数时，按逾期严重度（轻度 / 中度 / 重度）说明

### 4. 社交关系

- 综合解读 `contact_num` / `is_contact_black` / `bankcard_user_num`
- `is_contact_black = "是"` 必须明示为**重要风险信号**
- 联系人数量过低（如 0-1 人）需联动判断社交画像稀薄度

### 5. 规则命中

- 分别解读两条规则：`rule_hit_多头规则拦截` 与 `rule_hit_逾期未结清拦截`
- 哨兵值「无记录」必须明示为「规则未触发」，**不要**当作命中处理
- 命中状态必须明示触发的具体规则与建议跟进动作

## 严禁行为

- ❌ 输出 FICO 分数 / 信用等级 A/B/C/D / score_band
- ❌ 输出 account_type 列表 / 信用卡 / 房贷 / 车贷等具体账户类型
- ❌ 把哨兵值「无活体分」/「无逾期」/「无记录」当作有效数值参与计算或打低分
- ❌ 凭空虚构信用局类数据来源（如「国家信用局报告」）
- ❌ 输出超出 11 维特征以外的数值字段（避免幻觉）
