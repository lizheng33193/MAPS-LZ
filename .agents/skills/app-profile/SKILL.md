# App Profile

这个 skill 只服务于仓库里的 `App画像` 模块开发，不服务于运行时后端直接调用。

## 作用边界

- 适用于 `app/` 下的 App画像运行时代码、提示词模板、数据预处理、图表字段契约。
- 不适用于行为画像、信用画像、综合画像的业务逻辑。
- 不要把这个 `SKILL.md` 当成运行时 prompt 直接给后端模型调用。

## 运行时分层说明

- Codex 开发参考技能文件：`.agents/skills/app-profile/SKILL.md`
- 后端运行时真正调用的提示词文件：`app/prompts/app_profile_prompt.md`

二者允许共享业务规则，但代码侧只能读取 `app/prompts/app_profile_prompt.md`。

## 业务背景

你正在处理的是墨西哥本地化 App画像风控场景。目标是通过用户 APP 安装列表、首次安装时间、最后更新时间和分类标签，识别：

- 多头借贷风险
- 金融成熟度
- 消费能力与生活偏好
- 生活方式标签
- 最终风控建议

## 核心分类词典

| APP类别 | 代表APP举例（墨西哥市场） | 画像信号与业务推断 |
| :--- | :--- | :--- |
| 电商消费 | Mercado Libre, Amazon, Shein, Temu | 具备一定的消费能力，消费水平中等偏上至极高。 |
| 本地生活 | Rappi, Uber Eats, DiDi Food | 具备较高可支配收入，注重生活便利性与品质。 |
| 共享出行 | Uber, DiDi Rider, inDrive | 城市活跃人口，有日常出行预算和需求。 |
| 银行应用 | BBVA, Banorte, Santander, Nu | 银行化用户，金融成熟度高，通常具备较好的信用基础。 |
| 电子钱包 | Mercado Pago, Spin by OXXO, Ualá | 半银行化用户，习惯数字化支付，但不一定有传统银行账户。 |
| 借贷工具 | Kueski, Baubap, Tala, Moneyman, DiDi Préstamos | 有资金周转需求。若近期密集安装，代表极高的多头借贷风险。 |
| 政府/税务 | SAT Móvil, IMSS Digital | 极强加分项：代表有正规就业（有RFC税号/NSS社保号），纳税记录良好，资质优。 |
| 信用购物/BNPL | Nelo, Kueski Pay, Aplazo | 有信用消费习惯，多为年轻群体，接受超前消费。 |

## 分析规则

### 1. 时间权重

- 间隔 ≤ 7天：极高权重
- 间隔 8 - 30天：高权重
- 间隔 31 - 90天：中权重
- 间隔 91 - 365天：低权重
- 间隔 > 365天：极低权重

### 2. 多头借贷风险评估

- 高风险：近30天内密集安装 ≥ 3 个借贷 APP
- 中风险：近30天内安装 1-2 个，或历史安装较多但近期无新增
- 低风险：极少或未安装借贷 APP
- 必须输出明确判断依据

### 3. 金融成熟度与就业资质评估

- 输出：银行化用户 / 半银行化用户 / 非银行化用户
- 必须检查 SAT Móvil / IMSS Digital 等政府 APP，并在结论中体现正规就业/纳税信号

### 4. 消费能力评估

- 输出：高 / 中偏上 / 中 / 中偏下 / 低
- 结合电商消费、本地生活、共享出行、信用购物等类别判断

### 5. 生活方式标签

- 提炼 2-4 个标签

## 开发要求

- 优先修改 `app/prompts/app_profile_prompt.md`，而不是把业务内容只写在这里。
- `report_markdown` 必须与页面结构化字段一致。
- fallback 输出也必须遵守上述 Markdown 报告格式。
- 如果新增辅助脚本，区分“运行时脚本”和“Codex 开发辅助脚本”，不要混淆用途。
