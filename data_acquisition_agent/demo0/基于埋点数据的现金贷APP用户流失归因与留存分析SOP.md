这是一个创新且极具落地价值的分析框架。传统的数据分析往往只能看到宏观指标的下降，而我们通过“底层埋点数据 + 大模型微观理解 + 宏观总结归因”的路径，很好地填补了“从数据到用户真实行为动机”之间的空白。

为了让这个流程更加标准化、可复用，并方便团队成员的协作，现将流程梳理成了一份标准的 **SOP（标准作业程序）** 文档。

# **基于埋点数据的现金贷APP用户流失归因与留存分析SOP**

## **一、 文档目的与适用范围**

- **目的**：规范化利用大模型分析用户埋点数据的流程，精准挖掘现金贷用户的流失原因（归因），并产出可落地的业务优化策略，最终提升下单率、在贷规模等核心经营指标。
- **适用范围**：策略分析团队、用户运营团队、产品团队及风控团队进行客群留存分析及策略迭代时使用。 

## **二、 核心分析流程**

### **阶段一：业务逻辑梳理与挖掘目标定义**

**目标**：明确业务痛点，精准框定需要分析的目标客群。

1. **梳理业务漏斗**：复盘现金贷核心转化链路（如：注册 / register -> 完件（预审） -> 正审 -> 提现（下单）/ withdraw -> 放款 / grant -> 续贷）。

**重点转化点：**1.首贷后是否有续贷（mob0 -> mob1）；2.完件后是否下单；3.注册后是否完件（完件是指用户经过EKYC检测，填写基本信息，预审的过程，完件后展示offer，进入下单页面）

1. **定义流失客群**：根据业务周期（MOB, Month on Book）和转化漏斗，给出明确的流失客群定义。常见客群包括但不限于：

- **注册秒流失客群**：注册成功后，7天内未发起进件动作的用户。
- **MOB1 流失客群**：首月有借款行为，但在第一个账单周期后未复借的用户。
- **MOB4 流失客群**：历史留存较好（达到第4个月），但之后15天内突然无活跃或流失的用户。

1. **输出物**：目标分析客群定义及抽取口径

### **阶段二：埋点数据开发与样本抽取**

**目标**：为大模型准备格式规范、信息完整的原始原材料。

1. **样本抽取**：由于大模型存在Token限制及成本考量，无需全量跑批。针对每个客群，随机/分层抽取具有代表性的N个样例用户（初步按每个客群 50-200 个样本进行）。
2. **确认埋点范围**：根据定义的客群，提取其在流失前关键时间窗口内的所有行为埋点（如：页面停留时长、点击按钮、报错提示、退出动作等），需注意**避免出现数据穿越**。
3. **数据清洗与脱敏**：去除无效的冗余数据字段，当单用户的埋点数据量超过1000时对埋点数据进行截取，取最近的500条数据；**完成用户个人隐私的脱敏**，确保合规。
4. **输出物**：各客群样例用户行为轨迹数据集

### **阶段三：单样本大模型深度解析 (微观洞察)**

**目标**：利用大模型的逻辑推理能力，还原单个用户的真实操作情境与流失心理。

1. **构建 Prompt**：针对不同国家经营特点，设计专用的单用户分析 Prompt，要求大模型根据时间序列解读用户的操作意图、遇到的困难（如频繁修改信息、卡顿、阅读协议后退出等）。
2. **批量执行分析**：通过 API 批量将单用户的埋点数据输入大模型。
3. **生成独立报告**：为每一个样例用户生成一份详细的《个人埋点行为深度解析报告》。
4. **输出物**：数百份单用户的分析报告文本

### **阶段四：跨样本合并与归因提炼 (宏观总结)**

**目标**：从单体现象中寻找群体共性，对比不同群体的差异，得出最终归因结论。

1. **客群内聚类分析（找共性）**：

- 将同一客群（如：MOB1流失）的所有单用户报告合并到一个文件中。
- 使用总结类 Prompt 让大模型提取该客群流失的 **核心共性原因**（如：额度不及预期、利息感知过高、还款日提醒体验差等）。

1. **客群间对比分析（找差异）**：

- 将不同客群（如 MOB4未流失 vs. MOB4流失）的共性报告同时提交给大模型。
- 要求大模型分析差异点（例如：MOB4未流失多因为“对产品操作熟练”，而MOB4流失多因为“被竞品营销挖角/额度长期未提”）。

1. **输出物**：客群流失归因洞察及对比总结报告

### **阶段五：业务应用与指标追踪 (策略落地)**

**目标**：将分析洞察转化为具体的业务动作，并验证效果。

1. **策略制定**：联合产品与运营团队，针对提炼出的归因制定干预策略。

- *示例 1（产品层面）*：若注册流失客群集中在“活体检测”环节失败多次，需优化活体检测组件或增加引导提示。
- *示例 2（运营层面）*：若 MOB1 流失客群是因为“觉得利息高”，则针对此类客群在即将流失前下发“首期利息折扣券”。

1. **A/B 测试验证**：上线相关策略，设置实验组与对照组。
2. **效果回收**：追踪策略上线后的核心指标（如下单率、复借率），形成业务闭环。
3. **输出物**：业务优化方案及效果回收报告

## **三、 注意事项与最佳实践**

- **Prompt 迭代**：大模型的分析质量高度依赖于 Prompt。建立 **Prompt 库**，对“单用户分析”和“多用户总结”的提示词进行版本管理和调优。
- **幻觉控制**：大模型在总结时可能会产生幻觉。在“阶段四”抽查部分原始埋点数据，人工校验大模型得出的结论是否与底层数据相符。
- **数据安全**：提交给大模型（尤其是调用外部API时）的数据不包含用户的明文姓名、手机号、身份证号等敏感信息。
