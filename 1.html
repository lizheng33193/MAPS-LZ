一段话总结（面试压缩版）

Agent User Profile 是一套面向墨西哥/泰国现金贷的多智能体用户画像系统： 6 步管线（DataAccess → FeatureBuilder → DecisionEngine → Explainer → Reflection → Assembler） × 7 个画像 Skill（App / Behavior / Credit / Comprehensive / Product / Ops / Risk Decision）， 三权分立—— 规则引擎 决策可审计可回放， Vertex AI gemini-2.5-flash 把规则结论翻译成中文叙事， knowledge_agent RAG 从 630 chunk 中文风控指南召回行业背书； 支持 单 UID 个案分析（10 Tab Dashboard + KYC 尽调档案 .md 下载）和 Cohort 批量经营报表（双模式：⚡stats 纯规则 41ms/UID 服务 4 部门聚合 / 🤖full LLM 完整解读 110s/UID 服务客服个案）， 通过 NL Chat Orchestrator Agent 5 工具循环（含 NL→SQL ACK 安全门）让业务人员用自然语言对话触发任意分析路径， 底层验证 "LLM 不进批处理热路径" 的生产架构思想。

5 条主链路（数据流转过程）

A

单 UID Dashboard

~110s/UID 含全 LLM

受众：客服 / 审批员
浏览器 UID → DashboardView → 10 个 /analyze-module 并发预加载 → SkillRegistry 调度 7 skill → 每个 skill 走 6 步管线（规则 + LLM + RAG）→ 10 Tab + KYC 档案 .md

B

Cohort 双模式批量

⚡ stats 41ms / 🤖 full 110s

受众：风控 / 运营 / 产品 / 合规
/cohort-analyze {mode} → ThreadPool 5 并发 → stats: 5 skill 跳 explainer + NullModelClient 哨兵守门 + 4 视角聚合 + 模板摘要； full: 7 skill 全 LLM + executive summary → Excel/PDF 导出

C

NL Chat 5 工具循环

SSE 流式

受众：业务 PM 自然语言
/orchestrator/sessions → agent_loop ReAct 15 轮 → 5 工具循环：
parse_uid_file / run_profile / run_trace / run_cohort / query_data
query_data 走 NL→SQL → ACK 双确认 → pymysql 执行

D

RAG 风控决策证据

FAISS + BM25 + RRF

触发：risk_decision != approve
LLM 改写 query → FAISS 768 维 + BM25 jieba 中文 + RRF k=60 融合 → OCR 噪音过滤 → score 阈值过滤 → Lost in Middle 倒序 → 注入 LLM Prompt → RAG evidence badge "📚 引证 N 条"

E

深度报告导出（KYC 尽调档案）

3000-3500 字 markdown

受众：合规存档 / 客户尽调档案
Comprehensive Tab 玫红卡 / Trace Tab 紫色卡 → 下载按钮 → /api/{trace,comprehensive}/{uid}/detailed-report → Vertex AI generate_structured（max_output_tokens=24576）→ 6 章节模板（用户档案卡 + 数字基础设施 + 全生命周期 + 还款诊断 + 复贷博弈 + 战略总结）→ 失败 fallback 到 deterministic 简版

三权分立（架构核心 · 面试金句）

📐

规则引擎 · 决策权

if-else 阈值表 + country_pack 隔离
可审计 / 可回放 / 监管要求
5 个 decision_engine.py · 0 LLM 调用

🤖

Vertex AI · 表达权

gemini-2.5-flash · structured JSON
把规则代码翻译成中文叙事
让运营 / 客服 / 高管能直接读懂

📚

RAG · 知识权

FAISS 768 维 + BM25 + RRF + Contextual
从 630 chunks 召回行业最佳实践
给决策一份可追溯的证据背书

面试金句："决策权在规则求合规，表达权在 LLM 求可读，知识权在 RAG 求背书。 如果 LLM 把决策权也拿走了，金融业务就违反合规——第二天就被监管叫去喝茶。"
